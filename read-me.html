Problem Set 8 consist on the implementation of a page, namely Homepage, utilizing three main tools: HTML, CSS and JavaScript. To practice in this way the concepts relevant for week 8 of CS50's Introduction to Computer Science by Harvard. Within the specification section certain requirements have been set in place, in this 'Read me' page we'll go through the process of creating this website and the the fulfillment of the aforementioned requirements.

"1. The website must contain at least four (4) HTML pages"
I decided to make this website with topics related to my person and to CS50. I thought in different ways in which I could divide the sections into four main sections, each representing a different page in our site. Obviously, all the pages must be interconnected with each other in order for the user to navigate through the site. The pages were named as following:
Page HTML Identifier Description Additional Files & References
1. Landing Page index.html Homepage, with a brief general introduction of myself. styles.css, homepage.js, music.js
2. My CS50 Journey journey.html A revisiting of CS50's iconic problems, where I talk about my personal experiences. journey.css, bootstrap 5
3. Read Me read-me.html A step-by-step of the making of this project (pset 8) and the following of it's requirements. read-me.css, notes.txt, bootstrap 5
4. Contact Page contact.html Page with main social media accounts and contact information. contact.css, bootstrap 5

This requirement was very easy to fulfill since it occurs organically while completing the task in question. In spite of this I decided to list at least ten tasks I was completely certain I was going to use on my site. The list is as follow:
  1. <p>
  2. <img>
  3. <a>
  4. <div><span>
  5. <ol><ul><li>
  6. <button>
  7. <header><footer>
  8. <link><script><style>...
  9. <h1><h2>...
  10. <strong><em>...
  11. And many more!
"3. The website must integrate at least one feature from Bootstrap 5."
This 'read me' page is a testament of that implementation. The navigation bar, container, table and cards are all made using bootstap's classes.
"4. The website must have at least one stylesheet file of your own creation."
The landpage [index.html] was fully created by me without using bootstrap. That includes the Stylesheet and the JavaScript elements. The most difficult part was to make the page fully responsive, the initial design was created on web view just to realize at the end that it was a complete mess on mobile view. I used @media to implement the new solutions and I am very satisfied with the end result.
"5. The website must use at least five (5) different CSS selectors."
Like in our second requirement, this was very easy to achieve since it occurs organically while simply developing the project. Just to make sure I made a list with the different class selectors I was sure to implement. This is list is as follows:
  1. tag * (I applied general attributes for easier-to-understand margins and paddings)
  2. tag body, header, footer
  3. class .intro, .info, etc. (page is divided into sections each with it's own class)
  4. button, button:hover, button:active (I wanted to implement really cool buttons!)
  5. ID main-navigation (the collapse button requires an unique selector.)
"6. The website must use at least five (5) different CSS properties, such as font-size, or margin."
The CSS properties that I used the most were those that involved the position and size of the different elements. I learned how to use a display property called flex-box, this made my life easier when it came to placing each division in their corresponding position. My top 5 list of my most used properties in this project is as follow:
  1. display:
  2. justify-content: and align-items:
  3. width: & height:
  4. background-color:
  5. font-size:, color: & font-weight
"7. The website must integrate one or more features of JavaScript."
JavaScript, while confusing at first, is really satisfying to use once you get the flow going. While I used some bootstrap JS elements, my favorite part was to create scripts of my own. In 'homepage.js' I implemented the ability to go through different pictures and texts by the press of a button (in the landpage), also in 'music.js' I implemented a script to play music by clicking on a button! (also in the landpage).
"8. Ensure that your site looks nice on browsers both on mobile devices as well as laptops and desktops."
This was by far the most difficult requirement for me to follow. The reason is because I started the building process the wrong way! So, lesson learned, do not ignore the mobile view while designing your website. Always toggle between all relevant views to make sure of what you are doing. Luckily, I was able to make my site functional, while the mobile view is not perfect in terms of aesthetics, it is completely functional and it shows all of the elements the right way.
